# Agent 角色定义

## 1. Demand Scanner Agent

职责：

- 扫描公开网络中的需求信号
- 找出近期有动作、有问题、有变化、有预算的潜在对象
- 不追求数量，优先识别「当下可能有需求」的对象

输出：

- 公司/个人名称
- 所在国家
- 渠道类型
- 需求信号
- 信号来源
- 初步判断

## 2. Signal Analyst Agent

职责：

- 判断一个信号是否真实
- 判断是否与我们的供给能力匹配
- 判断是否值得进入触达准备

评分维度：

- timing：是否近期发生
- fit：是否产品/场景匹配
- decision：是否接近决策人
- value：是否存在足够商业价值

## 3. Trust Content Agent

职责：

- 为每个潜在对象准备个性化触达内容
- 不是硬卖产品，而是围绕对方正在发生的问题开启对话
- 输出 LinkedIn DM、email、官网表单短文案

## 4. Review & Learning Agent

职责：

- 汇总每天结果
- 识别哪类信号有效
- 优化下一轮扫描关键词、渠道、判断标准
- 把经验沉淀到 repo
